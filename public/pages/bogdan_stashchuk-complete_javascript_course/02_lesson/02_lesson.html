Урок 2: Переменные и типы

Переменные

Переменные дают возможность повторного доступа к значениям, которые присвоены этим переменным.

Именование переменных. Существует 3 подхода к именованию переменных в JavaScript
  • PascalCase - используется для того, чтобы называть типы и классы в JavaScript. Например есть такой класс как объект, либо array(массив) , они начинаются с заглавной буквы. Потому, что с помощью классов создаются экземпляры классов.
  • DB_PASSWORD - Такой подход к названию переменных используется тогда, когда значения, которые мы присваиваем переменным известны до запуска приложения и не меняются на протяжении работы приложения. Такие значения можно сказать константы. Например, такой переменной можно присвоить значение пароля или логина для доступа в базу данных. DB_PASSWORD DB_USER и т.д.
  • camelCase - такое именование переменных рекомендуется использовать во всех остальных случаях, и используется чаще всего.

Названия переменных должны быть понятными. В самом названии должна быть заключена информация относящаяся к сущности переменной. Например: 

                
                myName = "Сергей"

                myEmail = "sergeybabann@yandex.ru"
Лучше использовать название более длинным и понятным.

Объявление переменных

В JavaScript есть 3 ключевых слова с помощью которых объявляются переменные:
  • let
  • const
  • var
                
                let a // Объявление переменной с названием a
                
            
                
                const с = 10 // Объявление переменой с и присваивание ей значения 10
                
            
                
                a = true // Присваивание значения переменной

Переменной типа let можно присваивать новые значения, чего нельзя делать для переменной типа const
Переменной типа const должно быть присвоено значение сразу же при ее объявлении: 

                
                const c = 10
Переприсваивание значения переменной 
                
                let a = 10 // объявление переменной a и присваивание ей значения 10
                a = 20 /* присваивание нового значения переменной a, которая была объявлена ранее
                (новое значение число 20) */
Объявление переменной, и присваивание ей значение позже 
                
                let b // объявление переменной b
                b = false // присваивание переменной b значения false

Присваивание значения переменной происходит не сразу, а позже. Значение переменной b будет undefined, так как переменная объявлена, но никакого значения ей не присвоено.

Ошибка переприсваивания значения переменной типа const 

                const c = 10 // присваивание значения 10 переменной c
                c = 20 // нельзя менять значение переменной типа const, в консоли будет ошибка
Вывод переменной в консоль до ее обявления 

                console.log(a) /* в консоли будет ошибка с текстом
                script.js:1 Uncaught ReferenceError: a is not defined */

В тексте ошибки говорится о том, что переменная не определена. Мы пытаемся получить доступ к значению переменной, которой не существует. Она не была объявлена ранее. Для того, чтобы была возможность доступа к значению переменной необходимо эту переменную сначала объявить.

Объявление переменной и вывод ее в консоль 

                let a // объявление переменной a
                console.log(a) // вывод переменной a в консоль

После объявления переменной a вызываем log метод объекта console , и передаем в качестве аргумента название переменной a , и получаем в результате undefined потому, что значение переменной a после ее объявления будет undefined , так как мы еще не присвоили значение этой переменной.

Объявление переменной и присвоение ей значения 

                let a // объявление переменной a
                a  = true // присвоение переменной a значения true  
                console.log(a) // значение переменной true
Примитивные типы Типы переменных

Тип переменной определяется типом присвоенного этой переменной значения 


                const a = 10 // тип переменной число (number)

10 - это число. Тип number . Тип у этого значения - число, и когда мы присваиваем такое значение переменной a , то переменная a также будет иметь тип - число. Можно говорить, что переменная a - это число. 


                const b = 'abc' // тип переменной строка (string)

Присваиваем переменной b строку (string). Тип этой переменной строка. У переменной b тип будет строка (string). Можно говорить, что переменная b - это строка. Она содержит значение типа - строка.

Примитивные типы

Переменная имеет значение, значение имеет тип. То есть переменная имеет тип, после того как, мы присвоили переменной определенное значение. В JavaScript есть 6 примитивных типов:

  • string (строка)
  • boolean (логический), у него есть 2 значения: true и false
  • number (число), у него значения это числа
  • null, у него есть только одно значение - это null. Это значение используется тогда, когда мы явно хотим указать, что у какой-то переменной на данный момент значение отсутствует. Но мы явно это указываем самостоятельно. Мы можем присвоить значение null той или иной переменной.
  • undefined (значения нет, не существует), у него есть одно значение - undefined. Разница между null и undefined в том, что null присваивается именно мной самим той или иной переменной. Подразумевается, что значения для переменной пока нет, и возможно оно будет присвоено позже.
  • symbol (символ), с помощью такого типа можно создавать уникальные значения.
Ссылочный тип

Он всего один, и называется он object (объект)

Когда мы присваиваем одно из этих 4-ех значений примитивных типов какой-либо переменной, то в памяти компьютера эти переменные будут непосредственно содержать значения. Потому эти типы называются примитивные:

  • 'Hello, world!'
  • undefined
  • true
  • 25
Ссылочный тип Ссылочный тип это объект

У объекта есть различные значения. Если я присваиваю объект переменной, то переменная в памяти будет хранить только ссылку на этот объект. Переменная хранит ссылку на объект, который находится абсолютно в другом месте. Потому этот тип называется ссылочным.

Мы можем иметь несколько одинаковых ссылок в разных местах памяти, которые указывают на один и тот же объект. Тем самым я могу иметь 2 разных переменных и они содержат одно и то же значение. Одна переменная содержит это значение в одном месте памяти, а другая переменная содержит это же значение, но уже в другом месте памяти. Это разные переменные, но они содержат одинаковые ссылки. Эти ссылки ссылаются на один и тот же объект, который хранится в определенном месте в памяти. Можно создавать сколько угодно по количеству переменных, которые будут ссылаться на один и тот же объект. С примитивными типами такое невозможно потому, что значения примитивных типов хранятся непосредственно в переменных.

Если мы меняем значения каких-либо свойств объекта в одной из переменных, которая ссылается на объект, то эти значения изменятся и в других переменных, которые ссылаются на этот же объект. Меняя значения свойств объекта в какой-либо переменной, ссылающейся на этот объект, мы меняем значения свойств в самом объекте. А поскольку значения свойств объекта хранятся в одном месте в памяти, то изменения происходят во всех переменных ссылающихся на этот объект. Значения в свойствах объекта можно менять сколько угодно раз, независимо от того какому типу переменной был присвоен объект const или let .

Все это возможно благодаря тому, что объект это ссылочный тип. По сути, используя разные переменные, мы можем менять один и тот же объект. Ссылки при этом не меняются, но значения в самом объекте меняются.

Рассмотрим на практике 

                const objectA = { 
                            a: 10,
                            b: true
                        }

Создана константа objectA После такой команды в памяти будет создана переменная с названием objectA . Сам объект будет записан в другом месте памяти, а в переменную objectA будет записана ссылка на созданный объект. По сути это локация этого объекта в памяти компьютера. То есть objectA ссылается на объект в памяти. И у этого объекта 2 свойства со значениями.

Создадим копию переменной 

                const objectA = { 
                            a: 10,
                            b: true
                        }

                        const copyOfA = objectA // копия переменной objectA

Переменная objectA уже была объявлена ранее, и мы просто берем ссылку, которая была записана в переменную objectA и копируем ее в другую переменную, в переменную copyOfA . Объект остается в памяти только в одном месте, а ссылка на него копируется с одной переменной в другую переменную. Результатом этих двух инструкций будет то, что в памяти компьютера будет находиться 2 переменных copyOfA и objectA . Они будут содержать одни и те же ссылки на объект, который находится в другом месте памяти.

Изменение объекта 

                const objectA = { // объект 
                            a: 10,
                            b: true
                        }
                        
                const copyOfA = objectA // копия переменной objectA
                        
                copyOfA.a = 20 // изменение объекта

Что произойдет если мы попытаемся изменить объект? Изменить мы можем его, либо используя переменную objectA, либо переменную copyOfA потому, что они ссылаются на один и тот же объект. Доступ к свойствам того или иного объекта возможен с помощью оператора точка. Мы можем поменять значение или иного свойства с помощью такой команды: 


                copyOfA.a = 20

И в данном примере мы получаем доступ к свойству a , используя переменную copyOfA . Cвойство a имело значение 10 , мы присвоили ему новое значение 20 . Получили доступ к свойству a с помощью точки, и при помощи оператора = (оператора присваивания) изменили значение этого свойства. Также изменится сам объект. То есть по сути по ссылке, которая указана в переменной copyOfA мы пойдем найдем объект, который находится в другом месте памяти, найдем его свойство a , и поменяем значение этого свойства, которое указано в команде присваивания
copyOfA.a = 20 .

Из-за того, что объект у нас один и на него есть две ссылки, которые хранятся в разных переменных objectA и copyOfA , то если мы попытаемся получить доступ к свойству a объекта a вот таким образом: 


                copyOfA.a -> 20

то мы получим новое знчение 20 . Потому, что объект у нас один. Мы его поменяли через ссылку, которая хранится в переменной copyOfA, и тем самым если мы прочитаем какое-нибудь из свойств этого объекта через переменную objectA , мы получим новое значение, обновленное. Тем самым objectA.a = 20 будет равняться 20, и copyOfA.a -> 20 будет равняться 20 .

Это все происходит из-за того, что в JavaScript объект это ссылочный тип. Внутри переменных хранятся только ссылки. В данном примере ссылки одинаковы, объект у нас один, и мы можем менять объект, используя разные переменные.

Динамическая типизация

Папка 01 

                String a = 'abc'; 
                        // название типа значения

Название типа значения, которое будет присваиваться этой переменной. И этот тип String . И соответственно в данном примере мы присваиваем, как значение этой переменной строку 'abc' 


                int b = 10; // Мы указываем int

Мы указываем int , как префикс перед переменной, перед названием переменной. И в данном случае подразумевается, что тип переменной b будет integer . И мы соответственно присваиваем число этой переменной b . Тем самым перед объявлением переменной мы указываем какой тип значения эта переменная будет иметь. 


                String a = 'abc'; // название типа значения String, которое будет 
                присваиваться этой переменной

                int b = 10; // название типа значения int, которое будет 
                присваиваться этой переменной

                b = 'xyz'; // переменная типа строка // error, ошибка

String и int - это по сути ключевые зарезервированные слова. И если в таком случае вы попытаетесь присвоить значение другого типа, например переменной b , то вы получите ошибку. Потому, что значение переменной b должно быть обязательно integer . Код в этих примерах синтаксис не JavaScript. Этот пример кода - это пример статической типизации. Когда при объявлении переменной также указывается тип этой переменной.

В JavaScript нет необходимости указывать тип переменной, когда вы объявляете переменную, либо когда вы присваиваете ей значение. И следовательно в таком случае, если вы, например объявили переменную a (в JavaScript), присвоили ей значение типа 'строка', то позже вы сможете присвоить новое значение другого типа этой же переменной. И это будет выглядить так:

Папка 02. 

                a = 'abc' // String
                a = 10 // Number , присвоение переменой нового значения 10

Соответственно в JavaScript можно присваивать одной и той же переменной значения разных типов. Таким же образом, вы можете присвоить этой переменной, например, объект или массив, либо булевое значение. Что угодно. Это все возможно из-за того, что JavaScript динамически типизируемый язык. И возможность присваивать значения разных типов одной и той же переменной как раз вытекает из того, что такое динамиеская типизация.

Папка 03. Пример динамической типизации 


                        let a = 10 // number

                        a = true // boolean

                        a = 'Bogdan' // string

                        a = undefined // undefined

Происходит переприсваивание типов значений переменной a

Папка 04 

                function a() { 
                        console.log('Hey there')
                        } // это функция
                        
                        a(); // Это вызов функции 
                        
                        a = 10; // присвоение нового значения переменной а
                        
                        a(); // Uncaught TypeError: a is not a function
  • Здесь объявляется функция. function - это ключевое слово, a - это название функции.
  • Дальше мы можем вызвать эту функцию. Вызов функции происходит с помощью добавления скобок после названия переменной. Потом мы можем легко присвоить новое значение переменой a , на данном этапе эта переменная содержала в себе функцию. Функция была значением переменной.
  • Далее мы присваиваем новое значение переменной a
  • И если мы попытаемся снова вызвать функцию a , то мы получим ошибку с текстом: Uncaught TypeError: a is not a function
    На этом этапе a это уже не функция. Вместо этого a содержит примитивное значение типа - число. Вот как раз это недостаток динамической типизации - вы можете объявить функцию в одном месте, потом где-то позже случайно присвоить другое значение этой же переменной с таким же именем как вы объявили функцию, и все!, больше вы не сможете вызвать функцию, которую вы определили ранее, вы получите ошибку.

Какой выход из этой ситуации?

Выход очень простой! Вы должны использовать const везде, где это возможно. Тем самым вы предотвратите ситуации, когда вы случайно присвоите значение другого типа той или иной переменной. Потому, что const объявляет переменную и вы также должны присвоить значение этой переменной, все одной строкой. И позже новое значение присвоить нельзя.

Папка 05. const для объявления переменных

Рассмотрим пример. Вместо ключевого слова function применим немного другой синтаксис - он называется стрелочная функция. 


                const a = () => { 
                        console.log('Hey there')
                        } // это функция
                        
                        a(); // Это вызов функции 
                        
                        a = 10; // присвоение нового значения переменной а невозможно
                        
                        a(); // Uncaught TypeError: a is not a function

Стрелочная функция начинается со скобок, а между фигурными скобками ее тело. Присвоить новое значение переменной нельзя, потому, что она объявлена при помощи const , и поэтому после попытке присвоения нового значения переменной при ее вызове позже будет ошибка. Выполнение программы далее невозможно, и она не дойдет до вызова переменной.

JavaScript динамически типизируемый язык, и тип переменной не указывается, когда эта переменная объявляется, и основной борьбы с этим это использование const . Вы должны следить за тем, чтобы значения переменных не переприсваивались и не меняли свои типы.
